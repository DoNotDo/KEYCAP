# 와펜·키캡 재고 보고 가이드라인

카카오톡 그룹으로 받던 지점별 재고 보고를 KEYCAPS 앱에 맞춰 정리하고, **앱에서 보고받기**와 **출고에 유리하게** 쓰는 방법을 정리한 문서입니다.

---

## 1. 현재 카톡 보고 형식 정리

### 1.1 지점 구분

- **교보문고**: 평촌, 천안, 마곡, 영등포, 일산, 가든, 청량리, 건대, 목동, 판교, 광교, 강남, 수원, 동탄 등  
- **기타**: 부산(서면/센텀), 울산, 본사  
- 보고 끝에 지점명 표기: `-평촌`, `천안`, `교보문고 건대점`, `2월 18일 / 교보문고 영등포점` 등

### 1.2 와펜(파우치·에코백·목도리 등) 보고 형식

- **파우치 재고**: 품목별로 **색상/옵션별 수량**을 한 줄로 표기  
  - 예: `동전 파우치 빨강 2/주황 2/노랑 2/초록 2/파랑 2/보라 2`  
  - 예: `복주머니 파우치 소형 - 흰5/검7`, `대형 - 흰4/검2`
- **와펜(진열 품목)**: 니트백, 미니에코백, 캐시미어 목도리, 털 목도리, 장갑류 등 — 품목명 + 색상별 개수  
  - 예: `(미니에코백) 빨강-2개 / 하양-3개`
- **기타/하부장**: OPP봉투, 종이봉투, 스티커, 엽서, 건전지, 설명서 등 수량/묶음 단위

### 1.3 키캡 재고 보고 형식

- **하우징 구분**: 청축 / 갈축  
- **타입**: LED(흰/분홍), 정사각형 4구, 직사각형 4구, 직사각형 2구  
- **색상별 수량**: 하양, 보라, 파랑, 연두, 핑크, 하늘, 민트 등  
  - 예: `청축 LED 흰색 - 9개`, `4구 청축 하양 7/하늘 5/보라 7`
- **기타**: 종이봉투, 스티커, LED 건전지(CR1220), 포장지

## 2. 앱에서의 적용 원칙

### 2.1 품목 등록 규칙 (보고·출고에 유리하게)

- **카테고리**: 보고서에서 묶어서 보여줄 그룹  
  - 예: `파우치 재고`, `키캡 재고`, `와펜 하부장`, `키캡 하부장`
- **품목명**:  
  - **같은 품목의 색상/옵션은 별도 행으로** 등록하는 것을 권장  
  - 예: `동전 파우치 - 빨강`, `동전 파우치 - 주황`, …  
  - 또는 품목명에 옵션 포함: `LED 청축 흰색`, `정사각 4구 청축 보라`
- **지점**: 각 재고 행에 **지점명**을 지정하면, 지점별 재고 보고·출고 시 해당 지점만 필터링됩니다.

### 2.2 지점 목록

앱에 **와펜·키캡 보고용 지점 목록**이 등록되어 있습니다.  
재고 추가/수정 시 지점 선택에서 이 목록을 사용하면, 지점별 재고 보고와 출고 체크리스트에서 일관되게 활용할 수 있습니다.

- 교보문고: 평촌, 천안, 마곡, 영등포, 일산, 가든, 청량리, 건대, 목동, 판교, 광교, 강남, 수원, 동탄 등  
- 부산 서면, 부산 센텀, 울산, 본사

(실제 목록은 앱 내 **지점 > 지점별 재고 보고** 화면의 지점 선택 목록을 참고하세요.)

## 3. 앱 기능으로 보고받기

### 3.1 지점별 재고 보고 화면

- **위치**: 상단 탭 **지점** → **지점별 재고 보고**
- **사용 방법**  
  1. 지점을 선택합니다.  
  2. 해당 지점에 연결된 재고가 **카테고리별**로 묶여 표시됩니다.  
  3. **「카톡용 텍스트 복사」** 버튼을 누르면, 현재 보고 형식에 가깝게 정리된 텍스트가 클립보드에 복사됩니다.  
  4. 카카오톡 그룹 채팅에 그대로 붙여넣기 하면 됩니다.

### 3.2 보고 텍스트 형식 (자동 생성)

- 카테고리별로 그룹을 나누고,  
  - 같은 “대표 품목명”(예: 동전 파우치)끼리 **색상/옵션별 수량**을 `빨강 2/주황 2/...` 형태로 한 줄로 묶어서 생성합니다.  
- 품목명에 ` - `(공백-하이픈-공백)이 있으면, 앞부분을 대표명, 뒤를 옵션(색상 등)으로 간주해 묶습니다.

## 4. 출고에 유리하게 쓰는 방법

### 4.1 지점 관리·부족 현황

- **지점** 탭의 **지점 관리**에서 지점별 **부자재 부족 건수**를 확인할 수 있습니다.  
- 부족이 있는 지점을 누르면, 어떤 부자재가 얼마나 부족한지 상세 목록을 볼 수 있어, **출고할 품목과 수량**을 정하는 데 활용할 수 있습니다.

### 4.2 지점 발주 내역

- **발주** 탭의 **지점 발주 내역**에서 지점별·상태별(대기/처리중/출고/입고/완료) 주문을 관리합니다.  
- 출고 처리 시 **출고 수량**을 입력하면, 해당 수량만큼 완성재고가 차감되고, BOM에 따라 부자재 소모량이 자동 반영됩니다.  
- 지점별로 “이번에 보낼 것”을 정리할 때, 대기/처리중 주문을 필터해서 보면서 출고 리스트를 만드는 것이 좋습니다.

### 4.3 지점별 재고 보고와 출고의 연계

- **지점별 재고 보고**로 각 지점의 **현재 재고**를 카톡 형식으로 정리해 두면,  
  - 본사/물류에서 “어느 지점에 무엇이 부족한지”를 한눈에 보고,  
  - **출고 리스트**(이번 회차에 지점별로 보낼 품목·수량)를 만든 뒤,  
  - 앱에서 **지점 발주 입력 → 출고 처리**로 일관되게 기록할 수 있습니다.

---

## 5. 요약 체크리스트

| 목적 | 앱에서 할 일 |
|------|----------------|
| 지점 재고를 카톡으로 보고 | **지점** → **지점별 재고 보고** → 지점 선택 → **카톡용 텍스트 복사** → 카톡에 붙여넣기 |
| 출고할 품목/수량 정리 | **지점** → **지점 관리**에서 부족 현황 확인 + **발주** → **지점 발주 내역**에서 대기/처리중 주문 확인 |
| 출고 처리 기록 | **발주** → 지점 발주에서 **출고 처리**로 출고 수량 입력 |
| 품목이 보고서에 잘 묶이게 | 재고 등록 시 **카테고리**와 **품목명**(색상/옵션은 ` - ` 뒤에) 일관되게 사용 |
