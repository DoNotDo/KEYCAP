# 회사에서 이어서 작업하기

## 접속 주소
- **배포 사이트**: https://management-9f7d8.web.app
- **GitHub**: KEYCAPS 저장소 (최신은 `main` 브랜치)

## 로컬에서 실행
```bash
cd KEYCAPS
npm install
npm run dev
```
→ 브라우저에서 http://localhost:5173 접속 (실시간 반영)

## 배포 (수정 후 사이트에 반영)
```bash
npm run build
npx firebase deploy --only hosting
```
또는 `deploy-now.bat` 실행 후 메시지 입력해 커밋 + main 푸시 (GitHub Actions로 자동 배포 가능).

## 프로젝트 구조 요약
- **주간 재고 보고**: 상단 탭 **주간 보고** → 지점별 재고 수준(0~10)·판매 보고, 관리자는 현황/자동보고서/지점·품목 설정/생산·발주
- **생산·발주**: 주간 보고 탭 → **생산·발주** (관리자). 수요−완성재고=생산개수 → BOM으로 부자재 소요 → 발주요청서 생성
- **부자재 발주**: 발주 탭 → 부자재 발주 요약. 가로 막대 타임라인으로 발주일~입고일 표시
- **완성재고 ↔ 주간보고**: 재고 추가/수정 시 타입이 완성재고면 "주간보고 품목 연동" 선택 가능 (생산계획·BOM 연동용)
